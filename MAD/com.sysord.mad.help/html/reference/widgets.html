Widgets

Widget Categories

Output Text Widget

Widget for displaying read-only values, the default implementation is a non editable textbox.

Text Widget

Widget for editing string values, the default implementation is a textbox.

Boolean Widget

Widget for editing boolean values, the default implementation is a checkbox.

Number Widget

Widget for editing number values (Integer, Double, ...), the default implementation is a textbox.

Date Widget

Widget for editing date values, the default implementation is a textbox.

Single Select Widget

Widget for choosing a single value among several candidates proposed by a query. The default implementation is a combo.

Multi Select Widget

Widget for choosing multiple values among several candidates proposed by a query. The default implementation is list.

Navigation Widget

Widget proposing candidates for which it's possible to navigate toward them.

Flexible Widget

Widget for displaying a list of elements in function of their own configuration.

Xtext Editor

Widget for editing DSL elements with an embedded Xtext editor.

Custom Widgets

New Widget implementations can be added to the configuration by specifying their bundle and class qualified name.
All custom widgets are for a specified category of widget (see Widget Categories).

Declaration

To use a custom widget in a configuration, it must be defined by indicating its name, category, label, class qualified name and bundle symbolic name. 

	Widget type {
	    name: GOOGLE_MAP
	    category: OUTPUTTEXT_WIDGET
	    label: "Google Map"
	    custom widget: com.sysord.mad.ui.swt.widget.custom.GoogleMapWidget
	    from plugin com.sysord.mad.ui
	}

The configuration provided with the MAD core plugin already defines some useful custom widgets.

List

This is a single select widget implemented by an embedded list.

Example : Widget for the category property on a book. 

	widget: Category
	label: "Category"
	type: SINGLE_SELECT_WIDGET:List
	value: "category"
	candidates: "BookCategory.allInstances()"

Picklist

This is a multi select widget implemented by an embedded picklist.

Example : Widget for the property authors of a book 

	widget: Author
	label: "Author"
	type: MULTI_SELECT_WIDGET:Picklist
	value: "authors"
	candidates: "rootContainer().oclAsType(Library).writers"

Popup List

This is a single select widget implemented by a non editable textbox and a popup with a list.

Example : Widget for the category property on a book. 

	widget: Category
	label: "Category"
	type: SINGLE_SELECT_WIDGET:PopupList
	value: "category"
	candidates: "BookCategory.allInstances()"

Popup Picklist

This is a multi select widget implemented by a non editable textbox and a popup with a picklist.

Example : Widget for the property authors of a book 

	widget: Author
	label: "Author"
	type: MULTI_SELECT_WIDGET:PopupPicklist
	value: "authors"
	candidates: "rootContainer().oclAsType(Library).writers"

HTML Link

This is an output text widget implemented by an hyperlink.

Example : Widget displaying a hyperlink 

	widget: Link
	label: "Link"
	type: OUTPUT_TEXT_WIDGET:HTML_LINK
	value: "'http://www.sysord.com'"

Google Map

This is an output text widget using a string with coordinates for displaying a google map.

Example 

	widget: Map
	label: "Google Map"
	type: OUTPUT_TEXT_WIDGET:GOOGLE_MAP
	// Latitude : 43.975275, Longitude : 2.992272, zoom : 14
	value: "'43.975275::2.992272::14'"
