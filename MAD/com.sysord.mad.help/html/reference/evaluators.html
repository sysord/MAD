Evaluators

The Evaluators are responsible of the queries evaluations. They are referenced with an ID that must be specified as a language ID when writing queries.
All Evaluators must implements the interface QueryEvaluator.

Evaluator Declaration

The evaluators must be declared in the configuration by indicating their bundle symbolic name and their class qualified name.

Example : Declaration of the OCL evaluator 

	Query Evaluator {
	    language id: OCL
	    label: "Ocl evaluator"
	    Evaluator Class: com.sysord.mad.evaluator.impl.ocl.OCLQueryEvaluator
	        from plugin com.sysord.mad.core
	    description: "Default Query Evaluator"
	}

Out of the Box

The default MAD configuration provided by the core plugin registers several evaluators already implemented :

Evaluator Language ID Description
OCL OCL Evaluator for OCL queries.
Acceleo Acceleo Evaluator for Acceleo queries referenced in emtl files.
MAD MAD Evaluator with helpful queries for modifying EMF models.
Model Extension MODEL_EXTENSION Evaluator for model extensions.
